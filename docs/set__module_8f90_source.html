19 ! * WELCOME TO THE C O N T R O L & I N P U T MODULE IN XTB *
20 !! ------------------------------------------------------------------------
21 ! I use a singleton pattern for our input parser to ensure that you cannot
22 ! override your input data, therefore it is strictly necessary to follow a
23 ! strict hierachy on input sources. We decided that command arguments have
24 ! the highest priority, than follows the various input files and finally a
25 ! global configuration file is read in. The program structure is set up in
26 ! exactly that way, so this module has to behave like a singleton. Fortran
27 ! provides the save keyword, which is used in each set subroutine for this
28 ! purpose.
29 !! ------------------------------------------------------------------------
30 ! The syntax is:
46 ! unknown instructions are ignored and can be used to store additional info
47 ! that does not affect the outcome of the calculation
48 !
49 ! generally we distinguish between, ints, bools and reals. List's of arbitrary
50 ! combinations those can be archived by the `parse' subroutine from the MCTC lib.
51 ! Be carefully when reading arrays of unknown size, because you cannot backspace
52 ! on the input file. Multiline input is also not supported, but we have
53 ! arbitrary long lines to compensate for this.
54 !
55 ! To implement a new feature start by adding the variables to the header of
56 ! setparam.f90 (here), think about what you are doing, this module is integrated
57 ! in almost every important unit of xtb, so be careful when chosing variable
58 ! names (dum, nat, xyz and i usually bad ideas). You should use verbose names.
59 !
60 ! Next step is to define a default and add the printout in `write_set', add
61 ! the necessary information about the feature in the man page of `xcontrol'
62 ! and a short description in the manual. SERIOUSLY, do this first.
63 ! For all features that are incompatible with the singleton nature of `setparam'
64 ! use `constrain_param' and read the introduction there carefully.
65 !
66 ! Now you can add the necessary line in the `readflags' loop of `rdcontrol'
67 ! and define your own set subroutine here. If you add an option to an existing
68 ! instruction just add the case in the appropriate and set subroutine and
69 ! use a saved logical to guard it (I called them set<int>).
70 !
71 ! If you have any questions on this module, feel free to ask me or another dev
72 ! SAW: ehlert@thch.uni-bonn.de
73 !
